# NPS 工程図エディタ（NPS_pfcs）

工程図（Process Flow Chart）と部品別工程別能力表（Standardized Production Capacity Sheet）を、
ブラウザだけで作成・分析できるシングルページアプリケーションです。
ビルド不要・サーバー不要のプレーンな HTML / CSS / JavaScript で構成されており、
`index.html` を開くだけで動作します。

## できること

- **チャート**：工程記号（内製/外製・加工・検査・運搬・停滞）をドラッグ＆ドロップで配置し、
  線でつないで工程図を作成します。加工・検査記号にはメインライン基準で自動的に通し番号が振られます。
- **リスト**：工程を一覧形式で管理し、グループ分け・並べ替え・背骨グループ（メインライン）の指定ができます。
- **工程経路図**：複数の工程図を横断して、背骨グループ（またはグループ単位）の工程を比較表示します。
- **部品別工程別能力表**：工程ごとの加工時間・刃具交換時間・加工能力（個/日）・作業時間（手作業/自動送り）を
  一覧化し、必要数に対する能力の過不足をひと目で確認できます。
- **改善前 / 改善後モード**：同じ工程図に対して「改善前」「改善後」の2つのデータセットを切り替えながら比較できます。
- **保存・読込**：ブラウザの localStorage に自動保存されるほか、JSON ファイルへの保存・読込、
  印刷、PNG 画像出力（A4）にも対応しています。

## 使い方

ビルドツールや依存パッケージのインストールは不要です。以下のいずれかの方法で開いてください。

### そのまま開く
`index.html` をブラウザで直接開くだけで動作します。

### ローカルサーバーで開く（推奨）
一部のブラウザではファイルを直接開くとローカルストレージ等が制限される場合があるため、
簡易サーバー経由での起動を推奨します。

```bash
python3 -m http.server 8000
```

起動後、ブラウザで `http://localhost:8000/` にアクセスしてください。

## 基本操作

| 操作 | 内容 |
|---|---|
| 記号の配置 | 左パレットから記号をキャンバスへドラッグ、またはクリックして配置モードに入る |
| 記号の接続 | 記号の端子（ポート）同士をドラッグして線を接続 |
| フロー線への挿入 | 記号を既存の線の上にドロップすると、その位置に挿入される |
| 整列・チェック | ヘッダーの「整列・チェック」で記号間隔を揃え、ルール違反を検出 |
| 通し番号表示 | ヘッダーの「番号表示」で加工・検査記号への連番表示を切替 |
| 元に戻す/やり直す | `Ctrl+Z` / `Ctrl+Y`、またはヘッダーの「戻る」「進む」ボタン |
| 保存/読込 | ヘッダーの「保存」でJSON書き出し、「読込」でJSON読み込み |

## ディレクトリ構成

```
index.html      画面構造（HTML）
css/style.css   スタイル定義
js/core.js      定数・状態管理（記号定義・グループ・能力計算などの基礎ロジック）
js/graph.js     グラフ構造の計算（経路探索・通し番号の算出・線のルーティング）
js/render.js    SVGへの描画処理
js/ui.js        画面部品の生成・各ビュー（リスト/工程経路図/能力表など）の更新
js/main.js      イベント処理（ドラッグ&ドロップ・初期化・状態遷移）
```

スクリプトは `core.js → graph.js → render.js → ui.js → main.js` の順に読み込まれます。

## データについて

- 作業データはブラウザの localStorage に自動保存されます（別ブラウザ・別端末とは共有されません）。
- 複数の工程図をまたいで作業する場合や、他端末・他者との共有には JSON ファイルの保存/読込を利用してください。
